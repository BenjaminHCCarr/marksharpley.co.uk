Rio on K3s on a pi4

2020-01-08 :: Mark
#pi4  #rio  #k3s 

Say What?

Rio is a MicroPaaS for Kubernetes from Rancher. A not complete list of things Rio provides:

  • Automatic DNS
  • Load Balancing
  • Routing

We're going to be running it on a Raspberry Pi4, with Kubernetes coming from K3s, another Rancher project.

Installation

A few assumptions:

  • You have a working Raspberry Pi 4
  • A public IP and some way of routing traffic to your Pi.

Update your system:

sudo apt update && sudo apt install git

Install k3s

The k3s site does indeed suggest the below step - while not good for a production environment I'm okay with it for the purposes of messing around…

Curl a random script off the internet and pipe it to shell:

  curl -sfL https://get.k3s.io | sh -
  # Check for Ready node, takes maybe 30 seconds
  k3s kubectl get node
  # Kubeconfig is written to /etc/rancher/k3s/k3s.yaml

Install Rio

With k8s up and running, lets grab rio:

# Download the CLI (available for macOS, Windows, Linux)
$ curl -sfL https://get.rio.io | sh -   # or manually from https://github.com/rancher/rio/releases

# Setup your cluster for Rio
$ rio install

# Make sure all the pods are up and running. These may take several minutes.
$ rio -n rio-system pods

# Run a sample service
$ rio run -p 80:8080 https://github.com/rancher/rio-demo

# Check the status
$ rio ps
$ rio info
